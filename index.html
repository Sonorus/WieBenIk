Wie is Mike Roest?

Natuurlijk is mijn doel dus ook om deze opleiding af te ronden en een programmeur te worden. Voor de rest doe ik niet zo heel veel in mijn leven behalve wat games spelen of buiten rond hangen :)

Ik heb al aardig veel jaren en duizenden uren aan game ervaring, en kennis van computers zelf ontbreekt ook niet. Verder heb ik werk ervaring als vulploegmedewerker en afwasdienst in restaurants.

Waar is Mike eigenlijk mee bezig?

Dit zijn dingen waar ik eventueel naartoe wil werken.

  • Clean Code

    • Clean code is wanneer iemand een code schrijft en zorgt dat het er heel net uit ziet. Op deze manier kan iedereen jou code makkelijk overnemen en er goed op door werken. Er is geen vast protocol/manier van schrijven om het clean code te maken en elke ontwikkelaar heeft een persoonlijke kijk er op. Er zijn een aantal ideeën die worden beschouwd als de beste manier en dat vormt zo overzichtelijke codes binnen de industrie en de community, maar er is geen definitieve onderscheiding of manier waarop het moet. En ik denk niet dat er ooit zal zijn. Ik zou dit graag willen leren omdat ik op die manier snel een werkend project in elkaar zou kunnnen zetten, en het overzichtelijker werkt wat een hele hoop tijd scheelt.

  • Web Technologies

    • Web Technologies, het begrijpen en op de hoogte blijven van alles wat op dit moment gebruikt word en efficient werkt bij het ontwikkelen van wat dan ook op het internet. Het is belangrijk om bij te blijven met wat er op het moment nieuw en werkende is om het beste product te kunnen maken en leveren aan de klant. Up to date blijven is nooit verkeerd en daarom zou ik het ook willen leren.

  • Accessibility

    • Waar ben je bij het ontwikkelen als je zelf niet bij je bestanden, servers en projecten kan. Je moet altijd weten waar je verder kan werken, hoe je bij al je bestanden komt op elk moment. Ik wil elk moment bij alle bestanden kunnen die ik nodig heb en controle houden en daarom wil ik hier meer over leren, wat waarschijnlijk ook uitloopt tot andere gebieden in het vak.

  • Performance

    • Klant is koning, en de koning wilt luxe. Als je een klant hebt wil je natuurlijk dat die klant tevreden blijft en eventueel bij jou terug komt. Als jij de wens van de klant kan vervullen heb je vrij spel. Ook is het belangrijk als je website het blijft doen en de klant zo eenvoudig mogelijk alles kan uitvoeren op de site. Dit zou ik willen leren om in combinatie met alle andere skills all-round een goede programmeur te zijn.

  • Responsive Design

    • Wanneer je op je website op een knopje drukt, wil je ook dat het werkt. Als het werkt, wil je ook dat het er mooit uit ziet, en misschien wil je dingen ontwerpen die je niet ergens anders hebt gezien, dat een website mee werkt wanneer jij iets simpels uitvoert.

  • Game Development

    • Voor Game Development moet je heel veel beheersen, met name moet je goed communiceren en updates blijven geven. Samenwerken is cruciaal, ook kan wat creativiteit en ideeën uitbrengen zeker niets slecht betekenen in deze werkomgeving. Ook is verstand van de basics in games van belang.

  • Project Management

    • Project Management is een cruciaal onderdeel van elke baan als web developer. Als je goed weet hoe je een project gaat aanpakken en gaat indelen helpt het een stuk beter om alles bij te houden en garandeer je jezelf dat je project een success gaat worden. Ook houd je iedereen in de omgeving van je project tevreden.

  • Web Server Administration

    • Tenminste een klein beetje weten over de server waar je op werkt die jou website in de lucht houd helpt al snel met het oplossen van eventuele problemen en houd je website beter online. Veel ontwikkelaars denken dat het niet uit maakt hoe de server werkt, maar als je de server kent kun je ook makkelijker problemen opsporen en weet je hoe je server werkt.
